# Data Infrastructure Domain

## Description

Owns the entire data layer for CandleSight: PostgreSQL schema, Zero real-time sync, mutations, queries, database migrations, the queries-service backend infrastructure (app shell, CORS, rate limiting, health checks), and all core data endpoints (push handler, query handler, trade sync, spread stats, API key management, backtest job tracking, releases proxy). This domain does NOT own the AI proxy endpoints, Stripe webhook logic, or Clerk webhook logic -- those belong to ai-analysis and membership-payments respectively.

### queries-service/src/index.ts
Contains dependency wiring for ALL domains. When another domain adds a new dependency (e.g., a new webhook handler), they add it here and pass it to `createApp()`. This domain owns the file structure, migration startup, and the validated queries registration block.

## Primary Stack

- **TypeScript** -- all queries-service code
- **Hono** -- HTTP framework (lightweight, Web Standard APIs)
- **@rocicorp/zero** -- real-time sync framework (synced queries, push processor, mutations)
- **PostgreSQL** -- source of truth database (Railway hosted)
- **postgres.js** -- PostgreSQL client for migrations and direct queries
- **Zod** -- schema validation for mutation inputs
- **jose** -- JWT verification against Clerk JWKS
- **bcryptjs** -- API key hashing

## Key Dependencies

- **@rocicorp/zero/server** -- `PushProcessor`, `handleGetQueriesRequest`, `zeroPostgresJS`
- **@rocicorp/zero** -- `syncedQueryWithContext`, `withValidation`, `createSchema`, `createBuilder`
- **Clerk** -- JWT tokens signed with RS256/ES256, verified via JWKS endpoint
- **zero-cache** -- Railway service that provides WAL-based replication from PostgreSQL to clients
